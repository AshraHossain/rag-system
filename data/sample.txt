SECTION 2: PRODUCT PORTFOLIO
================================================================================

2.1 MeridianFlow (Data Pipeline Orchestration)
-----------------------------------------------
MeridianFlow is the company's flagship product, a cloud-native ETL and ELT pipeline
orchestration platform. It supports batch and streaming ingestion from over 300 data
sources including Snowflake, BigQuery, Redshift, Kafka, PostgreSQL, MySQL, MongoDB,
Salesforce, and SAP.

Key features:
- Declarative YAML-based pipeline definitions
- Auto-scaling compute on AWS, Azure, and GCP
- Built-in data quality checks with configurable SLAs
- Column-level lineage tracking
- Git-based version control for pipelines
- Visual DAG editor with drag-and-drop interface

Pricing: $0.10 per 1,000 pipeline task executions, with a free tier of 10,000 tasks/month.
Enterprise plans start at $5,000/month with dedicated support and custom SLAs.

2.2 MeridianSearch (AI-Powered Enterprise Search)
--------------------------------------------------
MeridianSearch is an enterprise semantic search product built on a hybrid retrieval
architecture combining BM25 keyword search with dense vector embeddings. It indexes
documents from SharePoint, Confluence, Google Drive, Slack, and Jira.

Key features:
- Sub-200ms query latency at 10 million document scale
- Cross-encoder reranking for precision improvement
- Role-based access control (RBAC) for document-level security
- Support for PDF, DOCX, PPTX, HTML, and Markdown
- API-first design with REST and GraphQL endpoints
- Explainable results with highlighted evidence spans

MeridianSearch uses a three-stage retrieval pipeline:
Stage 1 - Candidate retrieval: BM25 + approximate nearest neighbor (ANN) search returns
top-100 candidates. The dense model is a fine-tuned version of E5-large-v2.
Stage 2 - Reranking: A cross-encoder model (fine-tuned ms-marco-MiniLM-L-12-v2) scores
all 100 candidate pairs and returns top-10 results.
Stage 3 - Answer synthesis: An LLM generates a grounded answer with citations.

Pricing: $2,000/month for up to 500,000 indexed documents. Additional storage is
$0.002 per document per month.

2.3 MeridianLake (Data Lakehouse)
----------------------------------
MeridianLake is an open-format data lakehouse built on Apache Iceberg and Delta Lake.
It provides ACID transactions, time-travel queries, and schema evolution on object storage
(S3, Azure Blob, GCS). Integrated with Apache Spark, Trino, and Dask for compute.

Supports Z-ordering and Bloom filter indexing for query acceleration. Typical scan
performance is 3-10x faster than traditional Hive metastore setups.

3.1 Certifications

Meridian Technologies holds the following security certifications:

- HIPAA Business Associate Agreement (BAA) available
- FedRAMP Moderate (in progress, expected Q2 2025)

3.2 Data Encryption
--------------------
All data is encrypted at rest using AES-256 and in transit using TLS 1.3.
Customer-managed encryption keys (CMEK) are supported via AWS KMS, Azure Key Vault,
and Google Cloud KMS. Key rotation is automated on a 90-day schedule by default.

3.3 Access Control
-------------------
Meridian uses attribute-based access control (ABAC) with the following principles:
- Least-privilege by default: all new users have read-only access
- Multi-factor authentication (MFA) is mandatory for all accounts
- Single sign-on (SSO) via SAML 2.0 and OIDC is supported
- Session tokens expire after 8 hours of inactivity
- All API keys are hashed using SHA-256 before storage

3.4 Incident Response
----------------------
Meridian's incident response SLA:
- P0 (service outage): 15-minute response, 4-hour resolution target
- P1 (significant degradation): 1-hour response, 8-hour resolution target
- P2 (minor issues): 4-hour response, 48-hour resolution target
Security incidents are reported to affected customers within 72 hours per GDPR requirements.

SECTION 5: TECHNICAL ARCHITECTURE
================================================================================

5.1 Infrastructure
-------------------
Meridian's production infrastructure runs on Kubernetes (EKS, AKS, GKE) with
Terraform-managed provisioning. The control plane is multi-region active-active.
Data plane components are deployed per-region to comply with data residency requirements.

Supported regions: us-east-1, us-west-2, eu-west-1, eu-central-1, ap-southeast-1.

5.2 Data Residency
-------------------
Customers can select a primary region for data storage. By default, metadata (pipeline
configs, audit logs) is replicated to a secondary region for disaster recovery. Customers
can opt out of cross-region replication for strict data residency compliance.

5.3 Integrations and APIs
--------------------------
REST API: versioned at /api/v1, /api/v2. The v1 API will be sunset on March 31, 2026.
GraphQL endpoint: available at /graphql for all resource types.
Webhook support: HTTP POST with HMAC-SHA256 signatures for event-driven integrations.
SDK availability: Python (pip install meridian-sdk), JavaScript/TypeScript (npm install @meridian/sdk),
Java (Maven), Go (go get github.com/meridian-tech/sdk-go).

Rate limits: 1,000 requests/minute for Professional, 10,000 requests/minute for Enterprise.

5.4 Observability
------------------
All platform components export metrics in OpenTelemetry format.
Logs are structured JSON, forwarded to the customer's SIEM via Datadog, Splunk, or
Elastic integrations. Audit logs are immutable and retained for 7 years.

SECTION 10: FREQUENTLY ASKED QUESTIONS
================================================================================

Q: Can I bring my own LLM for MeridianSearch's answer synthesis?
A: Yes. As of version 3.8, MeridianSearch supports BYO-LLM via an OpenAI-compatible
API endpoint. You can point it to Azure OpenAI, AWS Bedrock, or any self-hosted model
that exposes an OpenAI-compatible interface.

Q: How does MeridianFlow handle schema drift?
A: MeridianFlow detects schema changes automatically using statistical fingerprinting.
When a new column appears, it is added to the destination schema if the pipeline is
configured with schema_evolution: "auto". If set to "strict", the pipeline pauses and
sends an alert.

Q: What is the maximum document size for MeridianSearch indexing?
A: Individual documents up to 50MB are supported. Larger documents are automatically
chunked into 500-token segments with 50-token overlap. The maximum index size per
tenant is 10 billion tokens (approximately 40GB of text).

Q: Does Meridian support on-premises deployment?
A: A self-hosted version is planned for Q3 2025. Currently, the platform runs exclusively
in Meridian's cloud infrastructure. Customers with strict data sovereignty requirements
can use the EU region with data residency guarantees.

Q: How is pricing calculated for MeridianFlow pipelines that run on a schedule?
A: Each task execution (one node in the DAG completing) counts as one task execution
unit. A pipeline with 10 tasks that runs hourly uses 240 task execution units per day
(10 tasks × 24 runs). The free tier covers 10,000 units/month, which supports roughly
14 such pipelines running hourly.

Q: What authentication methods does the API support?
A: The API supports API keys (Bearer token), OAuth 2.0 client credentials, and
short-lived JWT tokens (1-hour expiry). API keys can be scoped to specific resources
and actions using fine-grained permission policies.

Q: Is there a data export option if I want to leave Meridian?
A: Yes. Customers can export all pipeline configurations as YAML, all indexed documents
as JSON, and all audit logs as compressed CSV. Data export is available self-service
from the Settings > Data Portability section. For large exports (>1TB), contact support.
